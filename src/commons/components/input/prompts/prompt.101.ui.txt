==============================================
핵심요구사항) MCP를 연동하여 피그마의 디자인을 그대로 소스코드로 구현할 것.
                - 스토리북은 생성하지 말 것.
                - 기존의 공통컴포넌트와 스타일일관성을 통일하여 복합 컴포넌트(Compound Component)로 만들어낼 것.

핵심요구사항) 아래 요구사항을 따를 것.
            - Root 하위 3종(label / placeholder / addtional_text)을 활용하여 import·조합할 것.
            - htmlFor / id는 Context 기본값으로 연결하고, 소비자가 id·htmlFor를 넘기면 그 값을 우선으로 함.
            - AdditionalText의 state가 없으면 Field의 error·success·aria-invalid와 자동 동기화한다 (error→error, success→success, 그 외 default).
            - 입력 필드의 시각 상태·가상클래스·표준 속성·아이콘(showLeftIcon/showRightIcon) 규칙은 기존 프롬프트와 동일하다.

핵심요구사항) 입력 필드의 상태는 가상클래스(Pseudo-class)와 표준 속성으로 구현할 것. (JS로 `state`·variant 등을 고정하지 말 것)
            - default: 기본 상태
            - hover: :hover
            - focus: :focus 또는 :focus-within
            - disabled: :disabled, [aria-disabled='true']
            - error/invalid: [aria-invalid='true'] (필요 시 :invalid 보조)
            - success: .success 클래스 또는 [data-status='success']
            - filled: :not(:placeholder-shown)

핵심요구사항) 상태 우선순위·hover 제한은 하위 `placeholder`의 CSS 규칙과 일치시킬 것.
            - 우선순위: disabled > error/invalid > success > focus > hover > filled > default
            - disabled · error/invalid · success 구간에서는 hover 스타일이 겹치지 않도록 선택자로 제한할 것
            - disabled 상태에서도 `label` 색상은 기본값을 유지할 것

핵심요구사항) API는 HTML 표준을 우선할 것.
            - 루트는 `React.InputHTMLAttributes<HTMLInputElement>` 확장을 고려하고, `disabled`·`aria-invalid`·`data-status` 등은 실제 DOM에 반영할 것
            - 좌우 아이콘 노출은 `showLeftIcon` / `showRightIcon`(기본값 팀 규칙에 따름)으로 제어할 것
